# UI Theme Design Library

A deduplicated reference for designing themeable consumer apps, web products, dashboards, social experiences, and AI interfaces.

This library separates themes into four layers:

- **Foundation:** spacing, typography, hierarchy, density, and layout.
- **Visual personality:** color, materials, illustration, texture, and shape.
- **Interaction personality:** motion, gestures, storytelling, rewards, and adaptive behavior.
- **Context mode:** light, dark, accessibility, focus, seasonal, or event-specific presentation.

Use one foundation theme, one visual-personality theme, and optionally one interaction theme. Avoid combining several themes that solve the same visual problem.

---

# Theme Families

1. Minimal and editorial
2. Playful and human
3. Retro and expressive
4. Material and depth
5. Dark and futuristic
6. Interaction and behavior
7. Data and utility
8. Accessibility and personalization

# 1. Soft Minimalism

## Identity

Quiet, spacious, neutral, and highly usable. The interface feels calm rather than empty.

## Visual DNA

- Off-white or light-gray surfaces
- Large whitespace
- Thin dividers
- Low-contrast shadows
- Restrained iconography
- One muted accent color

## Typography

- Neutral sans serif
- Medium-to-large headings
- Comfortable line height
- Minimal weight variation

## Components

- Rounded cards
- Light borders
- Simple segmented controls
- Floating bottom actions
- Compact forms

## Motion

- Short fades
- Gentle scale transitions
- Minimal spring effects

## Best For

Productivity, finance, AI assistants, personal organization, and professional consumer apps.

## Avoid

Excessive decoration, neon colors, heavy gradients, and crowded dashboards.

# Recommended Theme Composition Model

Do not make every theme a complete replacement for the entire interface. Build the system from composable layers.

```ts
// Foundation controls layout, typography, density, spacing, and shape.
type FoundationTheme =
  | "soft-minimal"
  | "swiss"
  | "editorial"
  | "bento"
  | "data-dense"
  | "material";

// Personality controls visual expression.
type PersonalityTheme =
  | "playful-illustrative"
  | "zakka"
  | "korean-soft-luxury"
  | "organic"
  | "retro"
  | "y2k"
  | "neo-brutal"
  | "dark-premium"
  | "dreamcore"
  | "holographic";

// Material controls surface behavior.
type MaterialTheme =
  | "flat"
  | "glass"
  | "liquid-glass"
  | "neumorphic"
  | "clay"
  | "skeuomorphic";

// Interaction controls motion and behavior.
type InteractionTheme =
  | "minimal-motion"
  | "motion-first"
  | "story-driven"
  | "gamified"
  | "ai-native"
  | "conversational"
  | "ambient"
  | "voice-gesture";

// Accessibility controls user-specific presentation.
type AccessibilityTheme = {
  contrast: "standard" | "high";
  motion: "full" | "reduced" | "none";
  density: "comfortable" | "compact";
  textScale: "sm" | "md" | "lg" | "xl";
};
```

# Theme Selection Rule

A strong product theme should answer four questions:

1. **How should the product feel?**
2. **How dense should the interface be?**
3. **How expressive should motion and decoration become?**
4. **Which user controls must remain stable across every theme?**

Keep brand structure, navigation logic, accessibility, and task flow consistent. Allow themes to change emotional tone, visual materials, typography personality, illustration, and motion intensity.
